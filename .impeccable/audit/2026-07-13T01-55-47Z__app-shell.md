# Impeccable Audit: App Shell

Date: 2026-07-13
Scope: `app.js`, `index.html`, `styles.css`
Method: static implementation review, detector scan, existing automated tests

## Health Score

| Dimension | Score | Finding |
| --- | ---: | --- |
| Accessibility | 3/4 | Strong labels, focus styles, status messaging, and touch sizing; tab semantics and live-result announcements need tightening. |
| Performance | 3/4 | Lazy recipe/inventory images and modular rendering are good; image dimensions and loading-state strategy can improve layout stability. |

## Findings

### P1: Semantic tab state is incomplete

- Location: `index.html:191-196`, `inventory-ui.js:40-53`
- Category: Accessibility
- Impact: Shopping/Home controls expose `role="tab"` and `aria-selected`, but do not provide a complete keyboard tab pattern or explicit panel state. Screen-reader users may not get the same navigation model as sighted users.
- Recommendation: Add roving `tabindex`, keyboard arrow behavior, and synchronized `aria-hidden`/`tabindex` on panels, or use a simpler button-group pattern if tab semantics are not needed.
- Suggested command: `/impeccable harden grocery and inventory navigation`

### P1: Dynamic list changes are not consistently announced

- Location: `index.html:146`, `index.html:217-218`, `index.html:324-325`, recipe/inventory list render targets
- Category: Accessibility
- Impact: Counts, filters, empty states, and refreshed rows can change without a concise live announcement. Caretakers using assistive technology may not know that a filter or save changed the visible household state.
- Recommendation: Add scoped `aria-live` status summaries for result counts and action outcomes; avoid making entire long lists live regions.
- Suggested command: `/impeccable harden app feedback states`

### P2: Image dimensions are not declared at the markup boundary

- Location: `index.html:89`, dynamic recipe and inventory image templates
- Category: Performance
- Impact: Images can shift surrounding content while loading, especially on mobile where the fixed bottom navigation makes vertical movement more disruptive.
- Recommendation: Add intrinsic `width`/`height` or an aspect-ratio wrapper to all repeated image templates, including the Today backdrop.
- Suggested command: `/impeccable optimize images and loading states`

### P2: Design tokens have substantial implementation drift

- Location: `styles.css` throughout; detector reports repeated values outside `DESIGN.md`
- Category: Theming / maintainability
- Impact: Similar controls can diverge over time, bilingual wrapping fixes become harder to apply consistently, and future visual passes risk changing one surface while leaving another behind.
- Recommendation: Consolidate repeated type sizes, semantic colors, radii, and shadows into a small token layer, then migrate only repeated usages. Keep intentional one-off values documented rather than tokenizing every literal.
- Suggested command: `/impeccable extract styles.css`

### P2: Decorative texture competes with task content

- Location: `styles.css:27-31`, `styles.css:388-390`
- Category: Anti-pattern / performance
- Impact: Repeating stripe backgrounds are explicitly outside the product design rules and add visual noise behind dense grocery, inventory, and recipe content. They also make contrast evaluation less predictable.
- Recommendation: Replace the repeating textures with a restrained solid or single low-contrast tonal layer; reserve gradients for a clearly functional surface.
- Suggested command: `/impeccable quieter app background`

### P3: Browser-level responsive verification remains incomplete

- Location: `styles.css:2454-2564`, mobile form and filter layouts
- Category: Responsive design
- Impact: Static CSS has sensible breakpoints and safe-area handling, but narrow Spanish labels, browser-native selects, dynamic inventory rows, and increased text size still need visual verification in a real browser.
- Recommendation: Run a browser pass at 320px, 375px, 430px, tablet, and desktop widths with English and Spanish active; include keyboard focus and 200% text-size checks.
- Suggested command: `/impeccable live mobile app shell`

## Positive Findings

- The app has explicit product and design context in `PRODUCT.md` and `DESIGN.md`.
- Core fields use labels, localized accessible names, and visible focus styling.
- Mobile navigation includes safe-area padding and a reserved bottom content buffer.
- Reduced-motion overrides are present for transitions and animations.
- Recipe/source images use lazy loading, decoding hints, and escaped URLs.
- The codebase is modularized by workflow rather than leaving all UI behavior in `app.js`.

## Recommended Order

1. `/impeccable extract styles.css` to resolve the token drift documented above.
2. `/impeccable harden app feedback states` for live announcements and tab semantics.
3. `/impeccable optimize images and loading states` for layout stability.
4. `/impeccable live mobile app shell` once a local dev server is running.
5. `/impeccable polish app shell` as the final visual pass.

## Verification Limits

This audit was static because no dev server was running in the worktree. Full keyboard, screen-reader, contrast, and viewport behavior should be rechecked in a browser after the extraction pass.
